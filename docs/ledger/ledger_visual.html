Ledger kya hota hai?
Ek register jisme shop ka har lena-dena record hota hai — customer ne kitna kharida, supplier ko kitna dena hai, cash/bank mein kitna balance hai. Yeh sab ek jagah = LedgerEntry collection.
Double Entry System
Har transaction mein 2 entries banti hain — ek DEBIT, ek CREDIT.

Rule: Jo lega → DEBIT  |  Jo dega → CREDIT

Constants kyun?
Hardcoded strings likhne se typo ho sakti hai. Constants use karo taaki code safe rahe.

'debit' likhne ki jagah → LEDGER_ENTRY_TYPES.DEBIT
Model — enum, refPath, index
FieldKya karta hai
enumSirf allowed values accept karta hai — invalid values reject hoti hain
refPathDynamic collection — partyModel ki value dekhke Customer ya Supplier collection mein jaata hai
indexMongoDB queries fast karta hai — ek hi baar mein shopId + partyId + date filter
session parameter kyun?
Ek sale mein 2 entries ek saath banani hoti hain. Agar beech mein error aaye toh incomplete state na bane — isliye MongoDB Transaction use karte hain.

session hai → ya dono entries banegi ya koi nahi (Atomicity) ✅
getPartyBalance — Aggregation Pipeline
StepKya karta hai
$matchSirf us party ki active entries filter karo
$grouptotalDebit aur totalCredit sum karo — debit hai toh amount lo warna 0
$projectbalance = totalDebit - totalCredit calculate karo
reverseEntry — Kabhi Delete nahi
Ledger ka golden rule: entries delete nahi hoti — sirf reverse hoti hain. Isse audit trail bana rehta hai.

Galat DEBIT entry → CREDIT reverse entry banao → original ko REVERSED mark karo.
